# 战争前线 - KARDS风格双人联机卡牌游戏
# 项目设计书

## 📋 项目概述

**项目名称**: 战争前线 (War Front)
**项目类型**: 双人在线卡牌对战游戏
**参考游戏**: KARDS
**技术栈**: HTML5, CSS3, JavaScript (Node.js + Socket.io)
**目标平台**: Web浏览器
**开发模式**: 双人联机对战

## 🎯 项目目标

创建一个完整的KARDS风格二战题材卡牌对战游戏，专注于双人在线对战体验，实现：
- 完整的卡牌对战机制
- 实时双人对战
- 流畅的游戏体验
- 5个国家阵营系统
- 100+张卡牌

## 🎮 核心玩法设计

### 基本规则

1. **胜利条件**
   - 将敌方总部生命值降至0
   - 或在回合数达到上限时拥有更多总部生命值

2. **资源系统**
   - 指挥点(CP): 每回合自动增加，用于部署卡牌
   - 初始CP: 1点
   - 最大CP: 12点
   - 每回合CP增长: +1点

3. **回合流程**
   - 回合开始: 增加CP, 抽1张卡, 重置单位状态
   - 部署阶段: 使用CP部署单位到支援战线
   - 行动阶段: 移动单位、攻击敌人、使用指令卡
   - 回合结束: 切换到对手回合

4. **战场布局**
   - 前线: 4个可争夺的格子，控制前线获得优势
   - 支援战线: 每方各4个格子，用于部署单位
   - 总部: 玩家的生命值，可以被攻击

### 阵营系统

#### 5个主要阵营

**德国 (Germany) 🇩🇪**
- 特色: 强力坦克和装甲单位
- 加成: 装甲单位+1护甲
- 代表单位: 虎式坦克、步兵师、梅塞施密特战斗机
- 战术风格: 闪电战、装甲突击

**美国 (USA) 🇺🇸**
- 特色: 均衡发展，强力空军
- 加成: 空军单位+1攻击力
- 代表单位: 谢尔曼坦克、陆军步兵、P-51野马
- 战术风格: 制空权、火力压制

**英国 (Britain) 🇬🇧**
- 特色: 防守反击，皇家海军
- 加成: 单位+1生命值
- 代表单位: 丘吉尔坦克、皇家步兵、皇家海军
- 战术风格: 稳固防守、海军支援

**苏联 (Soviet) 🇷🇺**
- 特色: 人海战术，火炮支援
- 加成: 步兵单位-1费用
- 代表单位: T-34坦克、红军步兵、喀秋莎火箭炮
- 战术风格: 数量优势、火炮覆盖

**日本 (Japan) 🇯🇵**
- 特色: 轻型机动，特殊战术
- 加成: 单位+1移动次数
- 代表单位: 零式战斗机、帝国步兵、神风特攻
- 战术风格: 快速机动、突然袭击

#### 中立阵营

- 通用卡牌，任何阵营都可以使用
- 提供基础单位和功能性卡牌

### 卡牌系统

#### 卡牌类型（6种）

1. **单位卡 (Unit)**
   - 部署到战场上的战斗单位
   - 拥有攻击力、生命值、护甲
   - 可以移动和攻击
   - 示例: 步兵、坦克、战斗机

2. **指令卡 (Order)**
   - 立即生效的效果卡
   - 使用后直接进入弃牌堆
   - 示例: 闪电战指令、后勤支援

3. **增援卡 (Reinforcement)**
   - 直接获得单位到战场
   - 可以绕过费用限制
   - 示例: 乌拉冲锋

4. **压制卡 (Suppression)**
   - 降低敌方单位状态
   - 使敌人无法行动
   - 示例: 防空警报

5. **支援卡 (Support)**
   - 增强己方单位效果
   - 持续效果
   - 示例: 装甲增援

6. **反击卡 (Counter)**
   - 被动触发效果
   - 条件满足时自动生效
   - 示例: 伏击

#### 单位类型（8种）

1. **步兵 (Infantry)**: 基础地面单位
2. **坦克 (Tank)**: 强力装甲单位
3. **炮兵 (Artillery)**: 远程火力支援
4. **战斗机 (Fighter)**: 空中作战单位
5. **轰炸机 (Bomber)**: 对地轰炸单位
6. **舰船 (Ship)**: 海上作战单位
7. **防御工事 (Fortification)**: 固定防御建筑
8. **总部 (HQ)**: 玩家生命值目标

#### 关键字系统（15个）

**战斗关键字**
- **突击 (Blitz)**: 攻击时不受到反击伤害
- **反坦克 (Anti-Tank)**: 攻击坦克/装甲单位时伤害翻倍
- **对空 (Anti-Air)**: 攻击飞行单位时伤害翻倍
- **装甲 (Armor)**: 减少受到的伤害（伤害-护甲值）
- **轰炸 (Bombardment)**: 可以从任意位置攻击
- **先攻 (First Strike)**: 在敌人反击前造成伤害

**防御关键字**
- **支援 (Support)**: 相邻单位+1攻击力
- **烟雾 (Smoke)**: 无法被直接攻击
- **潜行 (Stealth)**: 攻击前无法被选中
- **防守者 (Defender)**: 被动时+1护甲

**部署关键字**
- **冲锋 (Rush)**: 部署后可以立即行动
- **坚守 (Hold)**: 前线单位死亡时移至前线
- **接力 (Relay)**: 死亡时让相邻单位行动
- **士气 (Morale)**: 提升相邻单位的战斗力

**特殊关键字**
- **独特 (Unique)**: 牌组中只能放1张
- **传说 (Legendary)**: 稀有度等级

### 牌组构建系统

**规则**
- 牌组大小: 40张卡牌
- 每种卡牌最多: 3张
- 阵营配置: 1个主阵营 + 1个次阵营（可选）+ 中立卡牌
- 牌组验证: 必须满足阵营限制和数量要求

**功能**
- 卡牌过滤（按类型、费用、阵营）
- 牌组统计（费用曲线、类型分布）
- 自动填充功能
- 保存/加载牌组
- 导入/导出牌组代码

## 🏗️ 技术架构

### 客户端架构

```
浏览器
├── HTML界面层
│   ├── 主菜单
│   ├── 牌组构建器
│   ├── 对战界面
│   └── 卡牌图鉴
├── CSS样式层
│   ├── 响应式布局
│   ├── 动画效果
│   └── 主题系统
└── JavaScript逻辑层
    ├── 游戏引擎 (Game Engine)
    ├── UI控制器 (UI Controller)
    ├── 网络客户端 (Network Client)
    └── 本地存储 (LocalStorage)
```

### 服务端架构

```
Node.js服务器
├── HTTP服务器 (Express)
├── WebSocket服务器 (Socket.io)
├── 房间管理器 (Room Manager)
├── 游戏逻辑服务器 (Game Logic Server)
├── 匹配系统 (Matchmaking System)
└── 数据存储 (可选: 数据库)
```

### 网络通信架构

**通信协议**: WebSocket (Socket.io)

**消息类型**
- 客户端→服务器
  - `join_room`: 加入房间
  - `deploy_unit`: 部署单位
  - `move_unit`: 移动单位
  - `attack`: 攻击
  - `end_turn`: 结束回合
  - `chat`: 聊天消息

- 服务器→客户端
  - `game_state`: 游戏状态更新
  - `player_joined`: 玩家加入
  - `player_left`: 玩家离开
  - `turn_start`: 回合开始
  - `effect_triggered`: 效果触发
  - `game_over`: 游戏结束

**同步机制**
- 客户端显示本地状态
- 服务器验证所有操作
- 关键操作需要服务器确认
- 使用操作序列号防止重复
