# 战争前线 - 联机对战服务器

## 📋 概述

这是战争前线卡牌游戏的联机对战服务器，使用 Node.js + Socket.io 实现实时通信。

## 🚀 快速开始

### 1. 安装依赖

在 `server` 目录下运行：

```bash
npm install
```

### 2. 启动服务器

```bash
npm start
```

或使用开发模式（自动重启）：

```bash
npm run dev
```

服务器默认运行在 `http://localhost:3000`

### 3. 启动客户端

在浏览器中打开 `index.html` 文件，然后：

1. 点击主菜单的"🌐 联机对战"
2. 点击"连接服务器"
3. 选择"创建房间"或"加入房间"

## 🔧 配置

### 端口配置

默认端口为 3000，可通过环境变量修改：

```bash
PORT=8080 npm start
```

### CORS配置

服务器默认允许所有来源访问，可在 `server.js` 中修改 CORS 配置。

## 📡 API

### HTTP API

#### 获取房间列表

```
GET /api/rooms
```

返回所有等待中的房间列表。

#### 健康检查

```
GET /health
```

返回服务器状态。

### Socket.io 事件

#### 客户端 → 服务器

- `create_room` - 创建房间
- `join_room` - 加入房间
- `leave_room` - 离开房间
- `player_ready` - 玩家准备
- `game_action` - 游戏操作
- `chat_message` - 发送聊天消息

#### 服务器 → 客户端

- `room_created` - 房间创建成功
- `room_joined` - 加入房间成功
- `room_updated` - 房间信息更新
- `player_joined` - 玩家加入
- `player_left` - 玩家离开
- `player_ready` - 玩家准备
- `game_started` - 游戏开始
- `game_state_updated` - 游戏状态更新
- `action_failed` - 操作失败
- `game_ended` - 游戏结束
- `chat_message` - 聊天消息
- `rooms_updated` - 房间列表更新
- `error` - 错误信息

## 🎯 使用示例

### 创建房间

1. 点击"连接服务器"
2. 输入玩家名称
3. 点击"创建房间"
4. 复制房间ID分享给朋友

### 加入房间

1. 点击"连接服务器"
2. 输入玩家名称
3. 输入房间ID
4. 点击"加入房间"

### 开始游戏

1. 两个玩家都进入房间
2. 各自点击"选择牌组"选择牌组
3. 点击"准备"按钮
4. 当双方都准备后，游戏自动开始

## 🔍 故障排除

### 无法连接服务器

1. 确认服务器已启动
2. 检查端口是否被占用
3. 确认防火墙设置
4. 检查浏览器控制台错误信息

### 房间创建失败

1. 确认已连接到服务器
2. 检查玩家名称是否有效
3. 查看服务器日志

### 游戏同步问题

1. 检查网络连接
2. 刷新页面重新连接
3. 查看浏览器控制台错误

## 📊 性能优化

### 房间清理

服务器会自动清理空闲超过30分钟的房间。

### 连接限制

服务器支持多房间并发，每个房间最多2名玩家。

## 🔒 安全注意事项

1. 在生产环境中，应使用 HTTPS
2. 实现身份验证和授权
3. 添加速率限制
4. 验证所有用户输入
5. 使用环境变量存储敏感信息
